# 系统设计与评分公式

## 架构

`app.py` 负责参与者流程和 session state；`src/` 分别负责数据校验、推荐、实验、解释、存储和统计；管理员页只展示匿名汇总和原始匿名表。系统不需要登录，不采集直接身份信息。

## 推荐算法

每套房源先尝试计算八个 0–100 分量：预算、目的地区域匹配、面积、地铁、租赁类型、装修、社区和安全。用户的 1–5 重要性转为非负权重；租赁类型使用固定权重 3。真实数据没有提供装修、社区和安全评分，这些维度保留为空，不进行虚构。最终分数只在该房源有真实数据的维度上重新归一化：

`score = Σ(weight_k × component_k) / Σ(available weight_k)`

预算在理想租金附近最高；超过最高预算后采用更陡惩罚。面积低于下限会额外下降；不接受合租时 shared 仅得 10 分。地铁距离每增加 25 米扣 1 分。位置匹配规则为：房源与参与者目的地同区100分、跨区40分、目的地未指定或房源区域缺失50分。它只表示行政区层面的代理匹配，不是通勤时间、距离或路线预测。所有可用分量及最终值均截断至 `[0,100]`。公式不含随机项；相同输入必得相同结果。

## 真实数据接入

`scripts/import_real_listings.py` 将用户提供的 29,006 条上海租赁挂牌记录转换为应用标准结构。它保留原始标题、位置、户型、面积、朝向、地铁描述和租金文本，并解析月租、面积、卧室、客厅和地铁距离。记录属于挂牌快照，不是成交数据；数据日期、可租状态和真实性无法从源文件验证。异常租金或面积记录仍留在归档中并标记为 `outlier_excluded`，但不进入实验抽样。

随机只用于参与者分组、选择集和展示顺序。分组通过带可配置盐值的 participant_id 哈希完成，保持参与者级固定且三组近似均衡。`EXPERIMENT_SEED` 可用于调试。

## 状态与容错

参与者数据、每轮选择、问卷使用唯一键 upsert；本地 CSV 用 UTF-8 BOM 写入。Supabase 不可用时自动回退本地。LLM 未启用、无密钥或请求失败时回退规则解释。刷新会保留浏览器会话状态；已提交记录用 `(participant_id, round_number)` 防重。
